# Spec 06 — Feed y UI

**Fecha:** 2026-06-13 · **Estado:** Aprobado · **Depende de:** Spec 01, 03, 04, 05

## Objetivo

La experiencia del egresado en React/Next + Tailwind: onboarding (conectar Gmail + subir
CV), feed con panel de impacto y match cards (badge salario, %, explicación), filtros y
página de perfil. Consume Elysia vía Eden Treaty.

## Alcance

- Login / landing con CTA.
- Onboarding: conectar Gmail → subir CV → sincronizar.
- Feed: panel de impacto + match cards + filtros + acciones.
- Página de perfil editable.

## Diseño técnico

### Páginas (App Router)
```
/                 landing + login Google
/onboarding       conectar Gmail (estado de /api/me) → subir CV (/api/profile/cv)
                  → botón "Sincronizar" (/api/ingest) con spinner → "Generar matches" (/api/match)
/feed             panel impacto + cards + filtros
/perfil           form editable (/api/profile)
```

### Panel de impacto (hero)
- Lee `/api/ingest/last`: *"Escaneamos 240 correos · 95 ruido filtrado · 18 empleos reales ·
  7 para TI"*.
- Refuerza el mensaje anti-ruido y anti-saturación.

### Match card
- Título · empresa · modalidad · ubicación.
- **Badge salario**: 🟢 `S/ 1500-1800 /mes` (token `success`) cuando `salario_explicito`;
  ⚪ `Sueldo no informado` (token `muted`) si no.
- `match 87%` (de `rerank_score`).
- *"Por qué te lo recomendamos: …"* (de `explanation`).
- Botones: **Postular** (abre `apply_link`), **Guardar**, **Descartar** → `PATCH /api/match/:id`.

### Filtros
- `solo con salario`, `modalidad`, `ubicación` → re-fetch a `/api/match?filter=...`.

### Estilo
- Tailwind v4 + tokens semánticos de `globals.css` (no colores hardcodeados).
- Estados de carga claros (ingesta/match pueden tardar): spinners/skeletons.
- Componentes presentacionales sin lógica de negocio (esa vive en server/hooks).

## Criterios de aceptación
- Flujo completo navegable: login → onboarding → sincronizar → matches → feed.
- El panel de impacto muestra números reales de `ingestion_runs`.
- Las cards muestran badge salario correcto y explicación por vacante.
- Guardar/Descartar actualiza la UI sin recargar y persiste.
- Filtro "solo con salario" oculta las vacantes sin remuneración.
- Sin colores hardcodeados; pasa `pnpm check`.

## Fuera de alcance
Digest (Spec 07). Animaciones avanzadas, settings extensos (YAGNI).
